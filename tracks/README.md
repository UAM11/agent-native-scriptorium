# tracks

这个目录用于承接长期主线、mission 和 workstream。

和 `journal/` 的区别：

- `journal/` 负责记录过程
- `tracks/` 负责组织长期推进的主线

和 `learning/` / `playbooks/` / `skills/` 的区别：

- `tracks/` 更像工作区
- 资产层更像沉淀后的长期成果

## 当前活跃主线

- `job-hunt-2026/`

## 轻量化原则

- 新主线优先先放一个 `README.md`
- 只有在材料明显增多时，再继续拆子目录
- 稳定资产应从 track 中晋升到其他长期目录，而不是永远留在 track 内
