# USER.md

这个文件记录的是用户的个性化偏好，而不是所有 agent 都必须永远遵守的通用规则。

## 作用

当未来有其他 agent 来维护这个仓库时，请优先把这里理解为“用户习惯与风格偏好说明书”。

## 当前偏好

- 根目录 `README.md` 使用英文。
- 可以同时维护一份中文伴随版 `README.zh-CN.md`，用于更自然地进行中文共建。
- 当仓库定位、结构或入口发生变化时，希望英文与中文两份 README 保持核心信息同步。
- 除 `README.md` 外，仓库内其余维护性文档优先使用中文。
- 每一份新的知识记录型 Markdown，默认都应包含元数据：
  - Created
  - Updated
  - Type
  - Status
  - Tags
  - Model
  - Harness
  - Source
- 当前默认值：
  - `Model: GPT-5.4`
  - `Harness: Codex`

## 双仓偏好

- 希望长期采用 `agent-native-scriptorium` 与 `agent-native-vault` 的双仓结构。
- 当前维护模式是“公仓优先”：默认只维护 `agent-native-scriptorium`。
- `agent-native-vault` 当前视为暂时冻结，除非用户明确要求使用私仓、恢复双仓协作，或发起一次同步请求。
- 用户可以按任务或阶段指定当前的主维护仓，不需要假设存在永久默认主仓。
- 除非用户明确发起同步请求，否则不要自动同步双仓。
- 用户的同步需求可以直接用自然语言描述，agent 应将其解析为主维护仓、目标仓、同步方式、包含范围、排除范围、覆盖策略。
- 也接受更短的口令风格，例如：`sync vault -> pub full keep-exclusive`
- 默认采用非破坏式同步：保留目标仓独有文件，不因源仓缺失而自动删除。
- 默认路由是：不确定是否适合公开的内容，先进入 `vault`，而不是直接进入 `scriptorium`。
- 经过脱敏、提炼、可被外部阅读和复用的内容，再晋升到 `scriptorium`。
- 如果同一主题同时有私密原稿与公开版本，公开版本优先保留结论和必要上下文，不复制全部私密细节。

## 对内容形态的偏好

- 如果内容更偏“人类阅读、理解、学习”，优先落为 `playbook`、`solution` 或 `learning` 笔记。
- 如果内容更偏“稳定步骤、重复复用、可直接交给 agent 执行”，建议进一步补一个 `skill` 版本。
- 如果内容是暂时还没做、但值得持续追踪的想法，可以先沉淀为 `idea` 条目。
- `idea` 不只用于产品方向，也可以用于记录科研、生活、创业、工作、日常等各种奇思妙想。
- 对于高价值工具型方案，优先采用“双轨制”：
  - 保留人类可读的知识条目
  - 同时补一个 agent 可复用的 `skill`
- 本仓库中的 `skill` 默认应遵循 Anthropic 官方的 agent skills 结构与最佳实践。

## 对仓库演化的偏好

- 希望这个仓库是长期沉淀型资产，而不是聊天记录转储站。
- 希望未来其他 agent 进入仓库后，能够快速读懂结构并继续维护。
- 希望每次有实质性改动时，都在 `CHANGELOG.md` 留下记录。

## 更新规则

当用户明确表达了新的个人偏好、风格习惯或维护方式时，请同步更新本文件。
